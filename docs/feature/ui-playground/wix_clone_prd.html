Site Builder — Visual Web Editor

Product: SiteBuilder Author: Claude (generated) Date: March 2026 Status: For Review
01 — Overview

Executive Summary

SiteBuilder is a browser-based, no-code visual website editor that lets users create and publish multi-page websites by dragging and dropping UI components onto a canvas — with no prior coding knowledge required.

Inspired by tools like Wix, the product differentiates by being AI-powered at its core: users can describe what they want and components are placed, configured, and styled automatically. Claude handles the intelligence layer in-browser, making it the fastest path from idea to live website.

Goal

Enable any user to build a functional, publishable website in under 10 minutes.

Non-goal

Not a code editor or developer tool. No raw HTML/CSS editing in v1.

Success Metric

≥60% of new users publish at least one site in their first session.

03 — Core Features

Feature Set (v1)

3.1 — Create Site Flow

The entry point. Users land on a dashboard that shows their sites. From here they can create a new site.

Create Site — Step by Step

Step 1 — Name your site. Free-text input. Auto-generates a subdomain slug (e.g. my-bakery.sitebuilder.io).

Step 2 — Pick a template or blank canvas. 6–12 curated starting templates per category (business, portfolio, blog, landing page). Or start blank.

Step 3 — Enter the editor. Canvas opens with the template or blank page loaded and ready to edit.

Requirement: the create-site flow must complete in ≤3 steps and <60 seconds.

3.2 — The Editor Canvas

The editor is the core product surface. It is a WYSIWYG canvas that mirrors what the published site will look like. Users interact with components by clicking, dragging, and using the property panel on the right.

Canvas behaviour requirements:

Live preview — changes are reflected immediately with no save/refresh cycle.

Grid-snap — components snap to an underlying 8px grid so alignment is consistent by default.

Multi-page — users can add pages (Home, About, Contact, etc.) and navigate between them in the editor.

Undo/redo — full undo/redo stack (Cmd+Z / Cmd+Shift+Z).

Responsive toggle — switch between desktop, tablet, and mobile previews.

3.3 — Component Library

The left sidebar contains a panel of draggable components. Users drag a component from the panel onto the canvas to add it. Each component is resizable and configurable via the right-side property panel.

Button

A clickable call-to-action element. Users can set label, link destination (URL or page), style (primary, secondary, outline), size, and corner radius.

Label text Link / href Variant Size Icon (optional)

Form

A configurable multi-field form. Users add/remove fields (text, email, phone, dropdown, checkbox, file upload), set a submission destination (email or webhook), and configure a success message.

Fields (drag to reorder) Submit destination Success message CAPTCHA toggle Required field toggle per field

Chat Window

An embedded AI chat widget. Visitors can type questions and receive automated responses. Powered by Claude. Site owners configure the system prompt, bot name, avatar, and colour scheme from the property panel.

System prompt Bot name & avatar Colour theme Powered by Claude API Conversation history toggle

Additional components out of scope for v1 but planned for v2: Video embed, Image gallery / carousel, Pricing table, Testimonials block, Navigation bar (auto-generated from pages), Map embed, Social feed widget.

3.4 — Property Panel

When a component is selected on the canvas, the right-side panel shows all configurable properties for that component. Changes apply in real time to the canvas. The panel is context-sensitive — it shows only properties relevant to the selected component type.

3.5 — Publish

A "Publish" button in the top bar deploys the current site to a free subdomain. Publishing is one click and completes within 10 seconds. Users can connect a custom domain (settings, not in the editor).

04 — Technical Architecture

System Design

Frontend

React + TypeScript. Canvas built with a drag-and-drop library (e.g. dnd-kit). State managed with Zustand.

Backend

Node.js + Express API. Sites stored as JSON schema in a Postgres database. Published sites rendered server-side to static HTML.

AI Layer

Claude API (claude-sonnet-4) powers the Chat Window component and the AI assistant sidebar.

Site Schema (JSON)

A site is stored as a JSON document. Each page is an array of component objects with position, size, and configuration properties:

{
  "siteId": "abc123",
  "name": "My Bakery",
  "pages": [
    {
      "pageId": "home",
      "title": "Home",
      "components": [
        { "id": "btn_1", "type": "button", "x": 120, "y": 240, "w": 160, "h": 44,
          "props": { "label": "Shop Now", "href": "/shop", "variant": "primary" } },
        { "id": "form_1", "type": "form", "x": 80, "y": 400, "w": 500, "h": 320,
          "props": { "fields": [...], "submitTo": "owner@example.com" } }
      ]
    }
  ]
}
