🎯 Key Findings & Recommendations

CRITICAL
POST /chat P95 collapses to 11.6s at high concurrency (100 VU)
The AI agent backend call becomes the dominant bottleneck. Under sustained high load, chat response times become unacceptable. Recommendation: Implement request queuing with backpressure, set per-user rate limits, and consider horizontal scaling of the agent worker pool.
CRITICAL
Session/auth layer fails at 50+ concurrent users sharing credentials
66.8% of POST /conversations requests fail with 403 at high concurrency. When multiple sessions share the same credentials, token refresh races cause cascading auth failures. Recommendation: Investigate token refresh locking strategy, implement per-session token isolation, and re-test with unique credentials per virtual user.
WARNING
POST /admin/banners shows 100% failure in destructive test
All 30 concurrent banner creation attempts returned 403. This is a permission issue — the test user lacks admin rights for this endpoint. Recommendation: Verify this is intentional RBAC enforcement. If admin users should be able to create banners concurrently, retest with proper admin credentials.
WARNING
Spike burst causes 42.9% POST /conversations failure
During the 50-user instant burst, nearly half of conversation creation attempts fail. While the server recovers within 15 seconds, real-world traffic spikes could impact user experience. Recommendation: Implement connection queuing or adaptive throttling at the load balancer level.
INFO
Memory leak test: STABLE — no leak detected
5-minute sustained load (3 workers, 4,668 requests) with 62 memory samples. Container memory stayed within 110.8–112.3 MiB (0.18% growth, R²=0.088). Memory released normally after load ceased. DB connections remained stable at 1–2. Follow-up: For production confidence, extend to 1-hour soak test with --duration 3600.
INFO
Volume handling is excellent
5KB messages with 20 messages per conversation show no performance degradation. The database and ORM layer handle large payloads efficiently.
INFO
Baseline and light-load performance are very good
At 1–5 VU, all endpoints respond within acceptable thresholds (<250ms P95 at baseline, <420ms P95 at 5 VU). The system is well-optimized for typical usage patterns.
